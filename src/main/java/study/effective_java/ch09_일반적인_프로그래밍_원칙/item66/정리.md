:book: 아이템 66: 네이티브 메서드는 신중히 사용하라


## 자바 네이티브 인터페이스
- 자바 코드가 **C/C++로 작성된 함수**를 호출하는 기술.
- **주요 쓰임**
  - OS/하드웨어/레지스트리 등 **플랫폼 특화 기능** 접근
  - **기존 네이티브 라이브러리** 재사용
  - **성능 개선 목적의 일부 구간 최적화**(꼭 필요한 부분만)

## 왜 신중히 써야 할까
- **타입/메모리 안전성 저하**: GC 관리 밖의 메모리, 포인터 오류, 메모리 누수 위험.
- **디버깅/유지보수 난이도 증가**: 예외/크래시가 JVM 전체에 영향을 미침
- **경계 비용(자바 <-> 네이티브 전환 오버헤드)**: 잦은 호출이면 오히려 느려질 수 있음.
- **코드 복잡도 증가**: 빌드, 스크립트, 로더, 보안 설정 등 주변 공정이 필요


## 핵심
- 네이티브 메서드를 사용하려거든 한번 더 생각하자.
- 네이티브 메서드가 성능을 개선해주는 일은 많지 않다.
- 사용하게 된다면 최소한만 사용하고 철저히 테스트하자.
- 네티이브 코드 안에 숨은 단 한개의 버그가 애플리케이션 전체를 훼손할 수 있다.

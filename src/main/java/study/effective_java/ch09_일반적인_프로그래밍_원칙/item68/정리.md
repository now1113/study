:book: 아이템 68: 일반적으로 통용되는 명명 규칙을 따르라

## 자바 플랫폼의 명명 규칙
- 자바의 명명 규칙은 **자바 언어 명세(JLS, 6.1)와 표준 라이브러리 관례**에 뿌리를 둔다.
- 규칙은 크게 **철자와 문법** 두 범주로 나뉜다.
- **특별한 이유가 없는 한 반드시 따르자.** 어기면 API는 배우기 어렵고, 사용/유지보수가 힘들어진다.


## 철자 규칙 (이름의 형태)

### 패키지/모듈
- **모두 소문자**, 요소를 `.`으로 구분해 **계층적**으로 구성.
- 일반적으로 **소유 도메인의 역순**을 접두로 사용: `com.google`, `edu.cmu`, `org.example`
- 단어를 **축약/혼용하지 말고** 짧고 명확하게 사용하자: `com.mycrop.util.collections.`

### 클래스/인터페이스/열거타입/애너테이션 타입
- **UpperCamelCase(PascalCase)**
- 너무 긴 이름 지양, 의미가 분명한 **명사/명사구**(인터페이스는 형용사도 가능)
  - ex) `Timer`, `HttpClient`, `List`, `Readable`

### 메서드
- **lowerCamelCase, 동사/동사구.**
  - ex) `size`, `add`, `put`, `remove`, `toString`, `hashCode`, `compareTo`

### 필드
- 일반 필드: **lowerCamelCase**
- **상수(의미상 불변, 공유 가능한 값)**: **UPPER_SNAKE_CASE**
  - ex) `DEFAULT_TIMEOUT_MILLS`, `MAX_VALUE`
  - 참고: `final`이라고 다 상수 네이밍을 사용하지는 않는다(의미상 불변일 때만)

### 열거 상수(enum constants)
- 관례적으로 **UPPER_SNAKE_CASE**: `SECONDS`, `HIGH_PRIORITY`

### 지역변수/매개변수
- **lowerCamelCase**: 짧은 범위에선 간명한 이름 허용

### 타입 매개변수(제네릭)
- **단문자 대문자** 관례:
  - `T`(Type), `E`(Element), `K`/`V`(Key/Value), `R`(return), `X`(Exception), `S`,`U`,`V`(보조 타입)

### 두 문자어/약어(Acronym)
- **단어처럼 취급**: 맨 앞만 대문자 -> `HttpClient`, `XmlStream`, `Id`, `Uuid`
  - `HTTPCLIENT`, `URLDECODER` 처럼 **전부 대문자는 지양**

## 문법(구문) 규칙

### 클래스/인터페이스 이름
- **명사/명사구**가 기본, 인터페이스는 경우에 따라 **형용사**도 허용.
  - ex) `List`(명사), `Comparator`(명사), `Readable`(형용사)

### 메서드 이름
- **동사/동사구**. 역할에 따른 관례:
  - **접근자/설정자(Beans)**: `getXxx`/`setXxx`, 불리언 접근자는 `isXxx`
  - **포함/여부**: `contains`, `isEmpty`
  - **변환**: **새 값 반환**은 `toType`(`toArray`,`toList`) / **뷰 제공**은 `asType` (`asList`, `asMap`)
  - **정적 팩토리(관례)**: `of`, `from`, `valueOf`, `getInstance`, `newInstance`, `newBuilder` 등
  - **표준 메서드**: `equals`, `hashCode`, `toString`, `compareTo`, `compare`

### 불리언 이름
- **긍정형 질문**으로 시작: `isEmpty`, `hasNext`

### 컬렉션/배열 관련
- 크기/길이/비어있음: `size`, `length`(배열 전용 필드), `isEmpty`
- 일괄 연산은 `addAll`, `putAll`, `removeIf`처럼 **의도가 드러나게**


## 주의
- **헝가리안 표기** 접두/접미사 남발 ex) `_value`, `mCount` 지양
- **구현 타입을 공개 API에 노출 금지**: `ArrayList`/`HashMap` 대신 `List`/`Map`등 **인터페이스 사용**
- **불명확한 축약어**, 사내 은어 사용 지양. **표준 단어를 사용하자**
- 상수도 아닌데 **UPPER_SNAKE_CASE** 사용 금지.

## 핵심
- 표준 명명 규칙을 체화하여 자연스럽게 베어 나오도록 하자
- 오랫동안 따라온 규칙과 충돌한다면 그 규칙을 맹종해서는 안된다. 상식이 이끄는 대로 따르자.

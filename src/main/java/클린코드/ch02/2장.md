## 2장: 의미 있는 이름

## 1. 이름은 `의도`를 분명히 드러내야 한다.

- 변수, 함수, 클래스 이름만 봐도 **무엇을 하는지 이해할 수 있어야** 한다.

```java
Commander c = CosmosContext.cosmosRequest.get().getCommander();     // X

if (!bbsCommt.getWrtrId().equals(c.getId())) {
    throw new IllegalArgumentException("");
}

Commander loginUser = CosmosContext.cosmoseRequest.get().getCommander();
String loginUserId =  CosmosContext.cosmosRequest.get().getCommander().getId();
```

## 2. 의미 있는 구체적인 이름을 사용하라

- **불분명한 추상적 단어**보다는 **구체적이고 명확한 단어** 사용

```java
List<String> targetProductNos  = new ArrayList<>(); // X
List<String> reviewTargetProductNos = new ArrayList<>(); // O
```

## 3. 이름에 불필요한 정보를 넣지 말라

- 이름에 `Info`, `Data` 처럼 **쓸모 없는 단어** 붙이지 않기
- **무조건 사용하지 말라는 의미는 아님**. `UserInfo`, `ProductData`처럼 **실제 의미가 다른 경우에는 예외적으로 사용해도 무방**

```java
List<SubscriptionInfo> subscriptionInfos = this.subscriptionInfoStore.queryAllByOrderItemIds(
        items.stream().map(Item::getId).collect(Collectors.toList()));  // X

List<SubscriptionInfo> subscriptions = this.subscriptionInfoStore.queryAllByOrderItemIds(
        items.stream().map(Item::getId).collect(Collectors.toList()));  // O
```

## 4. 검색하기 쉬운 이름을 사용하라

- 한 글자 이름(x, y, z)은 **짧은 스코프에서만 사용**
- `magic number` 대신 **의미 있는 상수 이름** 사용

```java
// 나쁨
for (int i = 0; i < 86400; i++) {...}

// 좋음
final int SECONDS_IN_A_DAY = 60 * 60 * 24;
for (int i = 0; i < SECONDS_IN_A_DAY; i++) {...}
```

## 5. 발음하기 쉬운 이름을 써라

- 코드는 동료와 **구두로도 소통** 해야한다.

```java
int genymdhms; // 발음 불가
int generationTimestamp; // 명확함

Commander c = CosmosContext.cosmosRequest.get().getCommander();
```


## 6. 의미가 중복된 단어를 제거하라 (말 줄이기)

> 같은 말을 반복하지 말자.

- `Product` 클래스 안에 `ProductName`, `ProductType`은 괜찮다.
- 근데 `ProductData` 클래스 안에 `productDataName`은 **불필요한 중복**

```java
public class Order {
    private Applicant orderer;          // O
    private Applicant orderOrderer;     // X
}
```

## 7. 코드의 문맥(Context)을 고려하라 (말 줄여도 되나?)

> 문맥이 있으니 말을 생략해도 된다.

- 클래스/모듈 이름이 맥락을 제공하면, 변수명은 간결하게

```java
public static class ClazzData {
    //
    private boolean useDelivery = false; // 배송여부
    private ClazzType clazzType;    // X
    private ClazzType type;         // O
    private boolean linkInserted = false; // 줌링크
    private List<LectureSchedule> lectureSchedules; // 강의 일정
}
```

## 8. 일관성 있게 작명하라

- 동일한 개념에는 동일한 단어 사용
- `get`, `fetch`, `retrieve` 섞지 말기
- 하나만 정해서 **전반적으로 일관성 있게**

```java
public Order queryOrder(QueryOrder query) {
    //
    return this.orderStore.retrieve(query);
}
```
통일
```java
public Order retrieveOrder(QueryOrder query) {
    return this.orderStore.retrieve(query);
}

```
```java
public Order queryOrder(QueryOrder query) {
    return this.orderStore.query(query);
}

```

## 9. 이름에 코드를 담지 말라

- 이름으로 변수 타입이나 구조를 드러내지 말 것(헝가리안 표기법 등)

```java
String strName;         // X
List<String> arrNames;  // X
String name;            // O
```

## 결론

- 의미 있는 이름은 코드의 **가독성과 유지보수성**을 극적으로 향상시킨다.
- 시간을 들여서라도 좋은 이름을 짓는 건 **투자 가치가 있음.**

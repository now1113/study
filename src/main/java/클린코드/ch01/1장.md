## 1장 핵심 요약: 깨긋한 코드란 무엇인가?

### 1. 깨끗한 코드란

- 깨끗한 코드란 **읽기 쉽고, 이해하기 쉬우며, 변경이 용이한 코드**를 의미
- 저자 로버트 C.마틴뿐만 아니라 여러 유명 개발자들의 관점을 함께 인용하며 설명


### 2. 여러 프로그래머들이 말하는 깨끗한 코드란?

#### Bjarne Stroustrup (C++ 창시자)
> 깨끗한 코드는 논리적이며 명료해야 한다. 깨끗한 코드는 한 가지를 제대로 한다.

#### Grady Booch (Object Oriented Analysis and Design with Application 저자)
> 깨끗한 코드는 단순하고 직접적이다. 깨끗한 코드는 잘 쓴 문장처럼 읽힌다.

### Dave Thomas (OTI 창립자)
> 깨끗한 코드는 단지 돌아가기만 하는 코드가 아니라, 읽는 사람에게 배려가 느껴지는 코드다.

### Michael Feathers
> 깨끗한 코드는 언제나 누군가 주의 깊게 짰다는 느낌을 준다.


#### 깨끗한 코드의 특징

- 한 가지 일을 잘 한다
- 이해하기 쉬운 이름을 사용한다
- 의도가 명확하게 드러난다
- 중복이 없다
- 작고 간결하다

### 깨끗한 코드의 좋은 예 vs 나쁜 예

- 좋은 코드: 읽기 쉽고, 수정하기 쉬움. **잘 쓴 에세이**처럼 읽히는 코드
- 나쁜 코드: 작동은 하지만, 이해하기 어렵고, 유지보수 하기 힘듦. 시간이 지날수록 기술 부채로 변함.

#### 더러운 코드
```java
public int c(int a, int b) {
    return a+ b;
}
```
#### 깨끗한 코드
```java
public int calcluateSum(int firstNumber, int secondNumber) {
    return firstNumber + secondNumber;    
}
```

### 깨끗한 코드가 중요한 이유

- 코드는 **한번 작성되고, 여러 번 읽힌다.**
- 팀의 생산성과 품질을 결정짓는 요소는 **읽기 쉬운 코드**다.
- 깨끗한 코드는 **버그를 줄이고, 변경을 두려워하지 않게 해준다.**
- 반대로, 더러운 코드는 **변경이 두려운 시스템**을 마는다.
